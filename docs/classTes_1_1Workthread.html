Detailed Description

The Workthread represents either a lightweight micro-thread or a system thread.

The Workthread can be compiled in two modes: micro-thread and system thread mode. Micro-thread mode executes using Unity's coroutine system, while system thread mode uses System.Threading.Thread. This allows platforms not supporting system threads to execute the same code using coroutines instead.

Because of the need to support coroutines, the Workthread executes a function via System.Collections.IEnumerator, just like Unity coroutines. This class simply wraps that enumerator and executes it. As such the function underpinning the enumerator must periodically yield return or potentially block the main thread.

A Workthread also requires a QuitDelegate and an optional StopDelegate. The former is called to gracefully when Stop(). The thread function enumerator must terminate on the next update and may no longer execute or risk blocking the system thread. The StopDelegate is called after the thread has been stopped and joined. Both are called from the main thread.

The thread function is normally called as fast as possible - at the frame rate for coroutines or at the CPU rate for system threads. If the work thread needs to sleep, it should yield return a CreateWait(float). This results in a sleep call for a system thread or a Unity WaitForSeconds for coroutines.

Finally the thread supports signalling via Wait() and Notify(). On calling Wait() a system thread will block until Notify() is called by another thread. Coroutines never block Wait(), so the return value must be checked.
